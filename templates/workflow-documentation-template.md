# Workflow Documentation Template

**Purpose**: Template for documenting processes and workflows in a timeless, step-by-step format.

## Structure

### 1. Overview (50-100 words)
- What the workflow accomplishes
- When to use it
- Key participants
- **Present tense** - describes current process

### 2. Prerequisites (50-100 words)
- What's needed before starting
- Required permissions
- Setup requirements
- **Concrete requirements** - not vague

### 3. Step-by-Step Process (200-400 words)
- Numbered steps
- Clear actions
- Expected outcomes
- Decision points
- **Complete steps** - not just "see guide"
- **Examples** for each major step

### 4. Verification (50-100 words)
- How to verify success
- What to check
- Common issues

### 5. Related Workflows (Links)
- Related processes
- Next steps
- **Supplement, don't replace** content

## Minimum Requirements

- **300+ words** substantive content
- **Complete steps** (not just summaries)
- **Examples** for complex steps
- **No placeholder text**
- **Clear outcomes** for each step

## Example Structure

```markdown
# Workflow Name

## Overview

[50-100 words on what it accomplishes]

## Prerequisites

[50-100 words on requirements]

## Process

### Step 1: [Action]

[Description of action]

**Example**:
```bash
# Complete command example
```

**Expected Outcome**: [What should happen]

### Step 2: [Action]

[Description of action]

...

## Verification

[50-100 words on how to verify]

## Related Workflows

- [Related Process](link)
```
