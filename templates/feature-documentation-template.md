# Feature Documentation Template

**Purpose**: Template for documenting system features in a timeless, concise, and dense format.

## Structure

### 1. Overview (50-100 words)
- What the feature is
- Why it exists
- Key benefits
- **No temporal language** ("recent", "new", dates)
- **Present tense** - describes what exists

### 2. Architecture/Implementation (100-200 words)
- How it works
- Key components
- Technical details
- **Not just links** - include actual content
- **Code references** where appropriate

### 3. Usage/Configuration (50-150 words)
- How to use it
- Configuration options
- Examples
- **Concrete examples** - not just "see example"
- **Complete code snippets** where applicable

### 4. Technical Details (100-200 words)
- Implementation specifics
- Performance characteristics
- Security considerations
- Integration points

### 5. See Also (Links)
- Related documentation
- API references
- Configuration guides
- **Supplement, don't replace** content

## Minimum Requirements

- **200+ words** substantive content (not counting links)
- **Concrete examples** (50+ words per example)
- **Technical details** (not just overview)
- **No placeholder text** (TODO, FIXME, XXX)
- **No vague descriptions**
- **Code references** with line numbers where applicable
